# Compiler Design Mini Project

A mini compiler front-end project built in Python that demonstrates the core phases of syntax analysis:

- Context-Free Grammar (CFG) generation
- FIRST and FOLLOW set computation
- Lexical analysis (tokenization)
- LALR parsing using PLY (`lex` + `yacc`)
- Step-by-step string parsing simulation for selected inputs

This project is designed for compiler design coursework and helps visualize how grammar-driven parsing works on a small C-like language fragment.

## Features

- Generates and prints the CFG used by the parser
- Computes and prints FIRST/FOLLOW sets in table format
- Tokenizes source code from `input.txt` with line/column output
- Parses the program using an LALR parser (PLY)
- Displays a sample LALR parsing table
- Accepts an interactive input string and shows shift/reduce steps

## Tech Stack

- Python 3.x
- [PLY](https://www.dabeaz.com/ply/) for lexing/parsing
- [tabulate](https://pypi.org/project/tabulate/) for pretty table output

## Grammar Overview

The grammar supports a compact C-style program pattern:

- Program entry: `int main() begin ... end`
- Declarations: `int x = 1;`
- Assignments: `x = x + 1;`, `n = n / 2;`
- While loop block: `while (n > 1) ... end while`
- Return statement: `return count;`

### High-Level Production Shape

```text
S -> DATATYPE FUNC LPAR RPAR BEGIN STMTBLOCK END
STMTBLOCK -> STMT STMTBLOCK | epsilon
STMT -> DECL | ASSIGN | WHILESTMT | RETURNSTMT
```

## How It Works

When you run `main.py`, the pipeline is:

1. Read source code from `input.txt`
2. Generate and print CFG productions
3. Compute FIRST and FOLLOW sets
4. Tokenize source text and print generated tokens
5. Run LALR parse on the same input
6. Prompt for a custom input string and simulate parsing steps

## Example Interactive Inputs

Use one of these when prompted:

- `id=id+1;`
- `n=n/2;`
- `return count;`
- `count=count+1;`

These are handled with predefined parsing traces in `parse_input_string()`.

## Sample Input Program (`input.txt`)

```c
int main()
begin
int count=1;
while(n>1)
count=count+1;
n=n/2;
end while
return count;
end
```

## Expected Output Sections

Running `main.py` prints:

- `GENERATED CFG`
- `FIRST AND FOLLOW TABLE`
- `TOKENS GENERATED`
- `LALR PARSER OUTPUT` (includes `Valid Program` for correct input)
- `LALR PARSING TABLE`
- `STRING PARSING USING LALR` (ACCEPT/REJECT result)

## Notes

- `parsetab.py` is auto-generated by PLY. Re-generate it by rerunning the parser.
- The epsilon symbol may display as `Îµ` in some terminals due to encoding; this does not affect parser logic.
- The interactive string parser currently supports specific hardcoded examples.

## Limitations

- Grammar is intentionally small and educational (not a full C grammar)
- Error reporting is basic
- Interactive parse traces are not dynamically generated for arbitrary grammar inputs
